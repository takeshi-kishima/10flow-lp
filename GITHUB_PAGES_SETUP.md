# GitHub Pages セットアップガイド

## 概要
このリポジトリは GitHub Pages を使用して自動デプロイされるように設定されています。

## 完了した設定

### 1. GitHub Actions ワークフロー
- `.github/workflows/deploy-pages.yml` を作成しました
- `main` ブランチへのプッシュで自動的にデプロイされます
- 手動トリガーも可能です (Actions タブから)

### 2. デプロイされるファイル
以下のファイルのみがデプロイされます：
- `index.html` - メインページ
- `css/` - すべてのスタイルシート
- `js/` - すべての JavaScript ファイル
- `assets/` - すべての画像とアイコン

除外されるファイル：
- `moto/` - 参考デザイン画像
- `test_offset.js` - テストファイル
- `.gitconfig` - Git設定ファイル

## 次のステップ

### GitHub Pages を有効にする

1. **リポジトリ設定に移動**
   - GitHub のリポジトリページに移動
   - `Settings` タブをクリック

2. **Pages セクションを開く**
   - 左サイドバーから `Pages` を選択

3. **ソースを設定**
   - Source: `GitHub Actions` を選択
   - これにより、ワークフローが自動的に Pages にデプロイします

4. **main ブランチにマージ**
   - 現在の PR を `main` ブランチにマージ
   - マージ後、自動的にデプロイが開始されます

5. **デプロイを確認**
   - `Actions` タブでデプロイの進行状況を確認
   - デプロイが完了すると、サイトが公開されます

### 公開 URL
デプロイが完了すると、以下の URL でアクセスできます：
```
https://takeshi-kishima.github.io/10flow-lp/
```

## トラブルシューティング

### ワークフローが実行されない
- リポジトリの Settings → Actions → General で Actions が有効になっているか確認
- `main` ブランチに変更がプッシュされているか確認

### ページが表示されない
- Settings → Pages で "GitHub Actions" がソースとして選択されているか確認
- Actions タブでワークフローが正常に完了しているか確認
- ブラウザのキャッシュをクリアして再度アクセス

### 画像やCSSが読み込まれない
- すべてのパスが相対パスになっているか確認（すでに対応済み）
- ブラウザの開発者ツールでネットワークエラーを確認

## メンテナンス

### サイトを更新する
1. 必要なファイル（HTML、CSS、JS、画像）を編集
2. 変更を `main` ブランチにプッシュ
3. GitHub Actions が自動的に新しいバージョンをデプロイ

### 手動でデプロイをトリガー
1. GitHub リポジトリの `Actions` タブに移動
2. `Deploy to GitHub Pages` ワークフローを選択
3. `Run workflow` ボタンをクリック
4. `main` ブランチを選択して実行

## 注意事項
- GitHub Pages は公開サイトです。機密情報を含めないでください
- デプロイには数分かかる場合があります
- 初回デプロイ後、DNS の伝播に時間がかかることがあります
