# 10FLOW ランディングページ

## プロジェクト概要

### 目的
10FLOWという自由自在のアップセルツールのランディングページを作成する。画像 `moto/1107.jpg` を参考に、シンプルで現代的なデザインを実装。

### 基本要件
- シンプルなページ構成
- ヘッダーは固定
- 仮画像を使用
- レスポンシブデザイン対応

## デザイン分析

### カラーパレット
- **プライマリ**: 青系グラデーション背景 (#1e3a8a → #3b82f6)
- **セカンダリ**: 白 (#ffffff) - カード背景
- **アクセント**: オレンジ (#f97316) - CTAボタン
- **テキスト**: 
  - メイン: 濃紺 (#1e293b)
  - サブ: グレー (#64748b)
  - 白文字: ヒーローセクション用 (#ffffff)

### タイポグラフィ
- **メインフォント**: Noto Sans JP (日本語対応)
- **フォールバック**: -apple-system, BlinkMacSystemFont, sans-serif
- **サイズ階層**:
  - H1: 48px (モバイル: 32px)
  - H2: 36px (モバイル: 28px)
  - H3: 24px (モバイル: 20px)
  - Body: 16px
  - Small: 14px

## ページ構成

### 1. ヘッダー（固定）
- **位置**: 画面上部固定
- **背景**: 半透明白 (rgba(255, 255, 255, 0.95))
- **要素**:
  - 左: 10FLOWロゴ
  - 中央: ナビゲーションメニュー
  - 右: 「資料請求」CTAボタン

### 2. ヒーローセクション
- **背景**: 青系グラデーション
- **レイアウト**: 左右2カラム
- **左側**:
  - メインタイトル: "自由自在の\nアップセルツール"
  - サブタイトル: 詳細説明文
  - CTAボタン: "今すぐ始める"
- **右側**: ダッシュボード画面のモックアップ

### 3. 機能紹介セクション
- **背景**: 白
- **タイトル**: "〜これらの課題がありませんか？"
- **レイアウト**: 5つのアイコン付き機能を均等配置
- **各機能**:
  - アイコン (64x64px)
  - タイトル
  - 説明文

### 4. 製品詳細セクション
- **背景**: 薄いグレー (#f8fafc)
- **構成**: 4つのサブセクション
  1. AIと人的リソースの組み合わせ
  2. ダイナミックプライシング機能搭載
  3. 人的対応による上質な顧客サポート
  4. ロイヤリティ向上による継続率向上
- **各セクション**: 画像（モバイル画面）+ 説明文

### 5. Platform Overviewセクション
- **背景**: 白
- **構成**: 料金プラン比較表
- **プラン**: 4つのプランカード
- **各カード**: 機能リスト、価格、CTAボタン

### 6. 会社情報・フッター
- **背景**: 濃紺 (#1e293b)
- **文字色**: 白
- **構成**: 
  - 会社概要
  - お問い合わせ情報
  - コピーライト

## レスポンシブデザイン

### ブレークポイント
- **Desktop**: 1024px以上
- **Tablet**: 768px - 1023px
- **Mobile**: 767px以下

### 各デバイスでの調整
- **Mobile**:
  - ヘッダーをハンバーガーメニューに変更
  - 2カラムレイアウトを1カラムに変更
  - 機能アイコンを縦並びに配置
  - フォントサイズ調整
  - タッチ操作対応（ボタンサイズ44px以上）

- **Tablet**:
  - 2カラムレイアウト維持
  - 機能アイコンを2-3列配置
  - 適切な余白調整

## 実装フェーズ

### Phase 1: 基本構造
1. HTMLセマンティック構造作成
2. CSS変数とリセットスタイル
3. 基本レイアウト実装

### Phase 2: スタイリング
1. カラーパレットとタイポグラフィ適用
2. 各セクションのスタイリング
3. コンポーネント作成

### Phase 3: インタラクション
1. 固定ヘッダー実装
2. スムーススクロール
3. モバイルメニュー

### Phase 4: レスポンシブ対応
1. メディアクエリ実装
2. 各ブレークポイントでの調整
3. 画像最適化

### Phase 5: 最終調整
1. パフォーマンス最適化
2. アクセシビリティ対応
3. ブラウザ間テスト

## パフォーマンス目標
- **First Contentful Paint**: < 1.5秒
- **Largest Contentful Paint**: < 2.5秒
- **Cumulative Layout Shift**: < 0.1
- **画像最適化**: WebP形式使用、遅延読み込み

## アクセシビリティ
- **WCAG 2.1 AA準拠**
- **キーボードナビゲーション対応**
- **適切なalt属性設定**
- **カラーコントラスト比4.5:1以上**
- **フォーカス管理**
